Build a smart parking management web app called "SurePark Baguio" using Next.js 15 App Router, TypeScript, Tailwind CSS, and shadcn/ui. The entire app uses a dark slate theme (bg-gradient-to-br from-slate-900 via-slate-800 to-slate-900). Use Inter from next/font/google.

========================================
TECH STACK
========================================
- Next.js App Router (TypeScript)
- Tailwind CSS + shadcn/ui
- lucide-react for all icons
- leaflet + react-leaflet for the map (loaded dynamically with ssr:false)
- localStorage for auth and slot data persistence
- Next.js Route Handlers for the ESP32 API layer
- postcss with both tailwindcss and autoprefixer plugins

========================================
DATA MODEL
========================================
ParkingSlot interface:
  id: number
  name: string
  location: string
  price: number
  status: "available" | "reserved" | "occupied"
  reservedBy?: string
  reservedAt?: number  (timestamp ms)
  paid?: boolean
  activeQrToken?: string
  checkedIn?: boolean
  bollardUp?: boolean  (true = raised/blocking, false = lowered/open)

5 default slots:
  { id:1, name:"Slot 1", location:"Session Road",  price:50, status:"available" }
  { id:2, name:"Slot 2", location:"Harrison Road", price:45, status:"available" }
  { id:3, name:"Slot 3", location:"SM Baguio",     price:60, status:"available" }
  { id:4, name:"Slot 4", location:"Cedar Peak",    price:40, status:"available" }
  { id:5, name:"Slot 5", location:"Mabini",        price:55, status:"available" }

Locations array: ["Session Road","Harrison Road","SM Baguio","Cedar Peak","Mabini"]

--- app/dashboard/page.tsx ---
"use client". Full feature dashboard. All state: user, slots, selectedLocation ("All"),
selectedSlot (ParkingSlot|null), paymentMethod ("GCash"), qrInput, scanResult,
showScanner (false), showMap (true), showTips (true).

HELPERS:
  syncSlots(updated) — calls setSlots + localStorage.setItem("surepark_slots")
  patchApi(slotId, patch) — PATCH /api/slots/:id, silently catches errors

ON MOUNT (useEffect):
  1. Read surepark_user from localStorage, if missing redirect to /login.
  2. Fetch /api/slots, on success syncSlots with result; on error fall back to
     localStorage surepark_slots or DEFAULT_SLOTS.

POLLING (useEffect every 3000ms):
  Fetch /api/slots with cache:"no-store". If result differs from current slots,
  update state + localStorage + keep selectedSlot in sync by id.

EXPIRY TIMER (useEffect every 1000ms):
  For any reserved slot where !checkedIn and now - reservedAt > 15*60*1000,
  reset it to available and call patchApi.

HANDLERS:
  handleLogout — remove surepark_user, push to /login
  handleReserve(slot) — patch status:"reserved", reservedBy:user.email, reservedAt:Date.now()
  handlePayment(slot) — generate qrToken = "SP-{id}-{Date.now().toString(36).toUpperCase()}",
                         patch paid:true, activeQrToken:qrToken
  handleScanQr() — validate qrInput against activeQrToken, check paid+reserved+not-expired,
                    patch status:"occupied", checkedIn:true
  handleReset() — POST /api/slots/reset, reset to DEFAULT_SLOTS, clear selectedSlot+scanResult
  handleBollardToggle(slot) — if !slot.paid return; toggle bollardUp; POST /api/bollard
                               with {slotId, bollardUp: !slot.bollardUp}
  handleCarLeft(slot) — if status!=="occupied" return; reset slot to bare available object,
                         POST /api/sensor with {slotId, carPresent:false}; setSelectedSlot(null)
  handleCarDetected(slot) — if status!=="reserved"||!paid return; patch status:"occupied",
                             checkedIn:true, bollardUp:true; POST /api/sensor {slotId, carPresent:true}
  getTimeRemaining(reservedAt) — returns "MM:SS" countdown string or "Expired"
  filteredSlots — slots filtered by selectedLocation (or all if "All")
  stats — {available, reserved, occupied} counts from filteredSlots
  myReservations — slots where reservedBy===user.email && status==="reserved"

========================================
API ROUTES (Next.js Route Handlers)
========================================

lib/store.ts — in-memory singleton:
  slotStore: ParkingSlot[] initialized from DEFAULT_SLOTS
  Export: getSlots(), updateSlot(id, patch), resetSlots()

app/api/slots/route.ts — GET returns all slots as JSON
app/api/slots/reset/route.ts — POST resets store to DEFAULT_SLOTS
app/api/slots/[id]/route.ts — GET returns single slot; PATCH merges patch into slot

app/api/sensor/route.ts — POST {slotId:number, carPresent:boolean}
  If carPresent=true AND slot.status==="reserved" AND slot.paid:
    Update slot: status="occupied", checkedIn:true, bollardUp:true
  If carPresent=false AND slot.status==="occupied":
    Reset slot to bare available (clear all reservation fields)
  Return updated slot as JSON.

app/api/bollard/route.ts
  GET ?slotId=N — returns {slotId, bollardUp: slot.bollardUp ?? false}
  POST {slotId:number, bollardUp:boolean} — updates slot.bollardUp, returns slot

========================================
IMPORTANT BEHAVIORS
========================================
1. Reservation expires after 15 minutes if not paid — auto-resets to available
2. After payment: slot stays "reserved" (not occupied yet)
3. Bollard control only unlocks after payment is completed
4. Car sensor (simulate or real ESP32) changes reserved → occupied
5. "Car Left" button resets occupied → available (clears all fields)
6. Dashboard polls /api/slots every 3 seconds so ESP32 hardware changes reflect live
7. All slot mutations write to both localStorage AND the API store
8. The map is placed BELOW the parking slots grid, not above
9. Map pins are large SVG teardrops (52px normal, 64px selected)
10. The entire app works offline using localStorage as fallback
